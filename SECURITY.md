# Security Policy

## Supported Versions

We currently support the following versions of PersistenceAI:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security seriously. If you discover a security vulnerability, please follow these steps:

### 1. **Do NOT** open a public issue

Security vulnerabilities should be reported privately to protect users.

### 2. Email us directly

Send details to: **PersistenceAI@proton.me**

Include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### 3. Response Time

We aim to:
- **Acknowledge** your report within 48 hours
- **Provide initial assessment** within 5 business days
- **Keep you updated** on progress

### 4. Disclosure Policy

- We will work with you to understand and resolve the issue
- We will credit you for the discovery (if desired)
- We will coordinate public disclosure after a fix is available

## Security Best Practices

When using PersistenceAI:

- Keep your installation up to date
- Review configuration files before use
- Use secure API keys and tokens
- Don't share sensitive information in logs
- Report suspicious activity immediately

## Security Updates

Security updates will be:
- Released as soon as possible
- Announced in release notes
- Tagged with security labels

Thank you for helping keep PersistenceAI secure!
